1. **M1 — Skeleton + Auth + User Management.** Monorepo, docker-compose Postgres, Spring Boot init, Flyway baseline (`app_user`, `audit_log`), Vite/MUI scaffold, `/login`, `/profile`, `/admin/users`, `/auth/**` + `/users/**`. Role enum includes FIRM_USER but firm linkage added in M2.
2. **M2 — Firms + Branches + Firm User scoping.** Tables `real_estate_firm`, `firm_branch`. Add `real_estate_firm_id` FK to `app_user` (with CHECK constraint). Endpoints `/firms/**`, `/branches/**`. `/admin/firms` screen. Firm-user creation flow in `/admin/users`. `/firm/deals` placeholder list.
3. **M3 — Deal CRUD + Broker Dashboard.** Tables `property`, `client`, `deal` (with `firm_branch_id` + POC fields). Deal endpoints (list/create/get/patch/delete/submit). 5-step `/deals/new` wizard starting with firm + branch + POC. `/my-deals`, `/deals/:id`. FIRM_USER scoping on `/deals` queries — `/firm/deals` now actually populates. Lifecycle so far: DRAFT, SUBMITTED.
4. **M4 — Document Upload (S3).** Real S3 bucket provisioned. `FileStorageService` + `S3FileStorageService`. `document` table (with OCR columns, all initially `NOT_APPLICABLE`). Upload-url / confirm / download-url / list / delete endpoints. `<DocumentUploader>` wired into wizard Step 4. OCR columns present but no OCR runs yet.
6. **M6 — Ownership Tree Backend.** Tables `ownership_structure`, `ownership_node`, `ownership_edge`. Full ownership endpoint surface. Cycle/percentage validation.
7. **M7 — Ownership Tree UI + Compliance Queue.** `/queue` (filters incl. firm/branch), `/deals/:id/review` skeleton, recursive `<OwnershipNode>`, node editor pane. Person-node creation in compliance pulls in OCR-extracted fields from any DL/passport attached to that node. Lifecycle: `assign` (SUBMITTED → UNDER_REVIEW).
